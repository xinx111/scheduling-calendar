## 1. 项目概述

### 1.1 产品定位
一款**排班管理 PWA**，最终打包为 Android APK。核心解决场景是：团队排班表通常以手写拍照形式在群内流传，本 App 让用户能拍照识别排班 → 快速生成个人日历视图 → 设置提醒。

### 1.2 核心功能一句话总结
> 拍一张手写排班照片 → AI 自动识别 → 生成多人的排班日历 → 选人切换查看 → 标记班次和同班同事 → 点日期写备注 → 闹钟提醒

### 1.3 目标用户
- 餐饮/零售/服务业从业人员（排班周期通常为一周）
- 需要同时查看自己和同事班次的团队成员

## 3. 数据模型

### 3.1 核心模型关系

```
排班模板 (ShiftTemplate)
    │
    ├── 定义了班次种类（早班/中班/晚班/休息等）
    │
排班记录 (ScheduleRecord) ──── 人员 (Person)
    │                                │
    ├── 某人在某天的某个班次         ├── 姓名、颜色标识
    ├── 同一天同班次 = 同班同事       │
    │                                │
备忘录 (Memo)
    │
    ├── 某天日期的备注内容
    ├── 可选提醒时间
    └── 可选闹钟标记
```

### 3.2 IndexedDB 数据表设计

#### Store: `persons` — 人员

```typescript
interface Person {
  id: string;           // 自生成 UUID
  name: string;         // 姓名（必填）
  color: string;        // 标识颜色 #xxxxxx
  avatar?: string;      // 头像 emoji 或首字母
  createdAt: number;    // 时间戳
  isActive: boolean;    // 是否在当前排班中
  order: number;        // 排序序号
}
```

#### Store: `shiftTemplates` — 班次模板

```typescript
interface ShiftTemplate {
  id: string;           // 自生成 UUID
  name: string;         // 班次名称（早班/中班/晚班/休息/通班）
  shortName: string;    // 简称（早/中/晚/休/通）
  startTime?: string;   // 开始时间 "08:00"（休息班次可为空）
  endTime?: string;     // 结束时间 "16:00"
  color: string;        // 显示颜色 #xxxxxx
  icon: string;         // 图标 emoji ☀️🌙🌄😴
  isDefault: boolean;   // 是否系统预置
}
```

**预置班次**（首次启动自动创建）：

| 班次 | shortName | 时间 | 颜色 | 图标 |
|------|-----------|------|------|------|
| 早班 | 早 | 08:00-16:00 | #F59E0B (琥珀) | ☀️ |
| 中班 | 中 | 12:00-20:00 | #3B82F6 (蓝) | 🌤️ |
| 晚班 | 晚 | 16:00-00:00 | #6366F1 (靛蓝) | 🌙 |
| 通班 | 通 | 08:00-22:00 | #EF4444 (红) | 🔄 |
| 休息 | 休 | — | #10B981 (绿) | 😴 |

#### Store: `scheduleRecords` — 排班记录（核心）

```typescript
interface ScheduleRecord {
  id: string;           // 自生成 UUID
  personId: string;     // 关联人员 ID
  date: string;         // 日期 "2026-05-29" 格式
  shiftId: string;      // 关联班次模板 ID
  source: 'ai' | 'manual'; // 来源：AI识别或手动录入
  createdAt: number;
  updatedAt: number;
}

// 索引: [personId + date], [date], [personId]
```

#### Store: `schedules` — 排班表批次（按周分组）

```typescript
interface Schedule {
  id: string;
  weekStart: string;    // 周一起始日期 "2026-05-25"
  weekEnd: string;      // 周日结束日期 "2026-05-31"
  title?: string;       // 可选标题 "5月第4周排班"
  createdAt: number;
  sourceImage?: string;  // 原始图片的 base64 或 blob key
}
```

#### Store: `memos` — 备忘录/提醒

```typescript
interface Memo {
  id: string;
  date: string;         // 关联日期 "2026-05-29"
  content: string;      // 备注内容
  remindAt?: number;    // 提醒时间戳
  isAlarm: boolean;     // 是否闹钟提醒
  isDone: boolean;      // 是否已完成
  createdAt: number;
  updatedAt: number;
}
```

### 3.3 关键查询场景与索引策略

| 场景 | 查询方式 | 索引 |
|------|----------|------|
| 某人的月排班 | 按 personId + date 范围 | `[personId+date]` |
| 某天的所有排班 | 按 date 查询 | `[date]` |
| 某天的同班同事 | 按 date + shiftId 查询 | (复合查询) |
| 即将到来的提醒 | 按 remindAt 范围 | (可加索引) |
| 按周查看排班 | 按 weekStart 查 schedules | `[weekStart]` |

### 4.2 模块详细说明

#### 4.2.1 拍照识别模块

**流程图：**
```
用户点击"拍照识别"
    ↓
打开相机 / 相册选择
    ↓
显示图片预览
    ↓
调用智谱 API（GLM-4V-Flash）
    ↓
返回结构化 JSON 数据
    ↓
展示识别结果表格
    ├── 用户确认 ✓ → 保存到数据库
    └── 用户修正 ✏️ → 编辑后保存
```

**AI Prompt 设计：**

```
你是一名排班表识别助手。请分析这张手写排班图片，提取出排班信息。

图片中的表格结构通常是：
- 行：人员姓名（左侧）
- 列：日期（周一至周日）

请按以下 JSON 格式返回：
{
  "weekStart": "2026-05-25",
  "persons": ["张三", "李四", "王五"],
  "shifts": [
    { "person": "张三", "date": "2026-05-25", "shiftName": "早班" },
    ...
  ]
}

班次名称规范：早班、中班、晚班、通班、休息
如果无法确定具体名称，请用最接近的。日期格式必须是 YYYY-MM-DD。
```

**容错机制：**
- 如果 AI 识别结果不完整，允许用户手动词格补全
- 支持对已保存的排班重新编辑
- 保留原始图片供对照

#### 4.2.2 日历视图

**月视图：**
- 顶部：年月切换（左右箭头）+ 今日按钮
- 人员选择器（横向滚动，当前选中高亮）
- 网格日历，每个日期格子显示：
  - 日期数字
  - 班次标签（带颜色圆角条）
  - 同班人数标记（如 "👥 3人"）
- 点击日期 → 进入日详情

**日详情：**
- 上半部分：当日排班卡片
  - 当前选中人的班次（大图标+时间）
  - 同班同事列表（头像+名字）
- 下半部分：备忘录区域
  - 已有备注列表
  - 添加备注按钮

#### 4.2.3 提醒系统

**PWA 阶段：**
- Service Worker 后台通知（需用户授权）
- 页面内弹窗提醒（使用 Dialog 组件）
- 提醒列表统一管理

**APK 阶段（Capacitor）：**
- 使用 Capacitor Local Notifications 插件
- 支持闹钟样式提醒（震动+声音+通知栏）
- 即使 App 在后台也能弹提醒

## 6. AI 识别流程

### 6.1 识别步骤

```
Step 1: 用户拍照/选图
Step 2: 前端压缩图片（最大 1024px 宽）
Step 3: 调用智谱 GLM-4V-Flash API
        方法: POST https://open.bigmodel.cn/api/paas/v4/chat/completions
        body: { model: "glm-4v-flash", messages: [{
          role: "user",
          content: [
            { type: "image_url", image_url: { url: base64_image } },
            { type: "text", text: prompt_text }
          ]
        }]}
Step 4: 解析返回 JSON
Step 5: 匹配已有人员（按姓名模糊匹配）
Step 6: 匹配班次（按名称匹配 ShiftTemplate）
Step 7: 展示识别结果供用户确认
Step 8: 用户确认后，按周写入 IndexedDB
```

### 6.2 容错与纠错

| 问题 | 处理方式 |
|------|----------|
| AI 返回的姓名不匹配已有人员 | 自动创建新人员，标记为"待确认" |
| 班次名称不匹配（如"早"vs"早班"） | 模糊匹配，最接近的班次 |
| 某天识别为空 | 标记为"休息"或留空让用户填写 |
| 整张图片无法识别 | 提示用户重拍（确保光线充足、表格清晰） |

### 6.3 数据导出（可选增强）

- 导出为 CSV 或 iCal 格式
- 分享排班给他人

## 7. APK 打包方案

### 7.1 方案对比

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| **PWABuilder** | 在线工具，输入 PWA URL 即可生成 APK | ✅ 最简单，❌ 功能受限，不能访问原生 API |
| **Capacitor** | Ionic 团队维护，PWA → 原生壳 | ✅ 能调用原生 API（通知、闹钟、震动），❌ 需要 Android Studio |
| **TWA (Trusted Web Activity)** | Google 官方方案 | ❌ 配置复杂，不建议 |

**推荐方案**：先用 PWA 开发和测试，功能稳定后用 **Capacitor** 打包为 APK，这样可以调用原生通知和闹钟 API。

### 7.2 Capacitor 集成计划

```
npm install @capacitor/core @capacitor/cli @capacitor/android
npm install @capacitor/local-notifications  // 本地通知/闹钟
npx cap init
npx cap add android
```

**需要的原生插件：**
- `@capacitor/local-notifications` — 本地通知和闹钟
- `@capacitor/device` — 设备信息
- `@capacitor/filesystem` — 文件存储（保存原始排班图片）
